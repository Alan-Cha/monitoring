# 預先定義的儀表板
{: #default_dashboards}
您可以選擇下列任何預先定義的儀表板來監視基礎架構：
{:shortdesc}


## 應用程式
{: #default_dashboards_applications}

下表列出您可以選取來監視應用程式及基礎架構元件的預先定義儀表板：

| 儀表板名稱        | 說明                                                            | 用法             |
|-----------------------|------------------------------------------------------------------------|-------------------|
| `HTTP 概觀`         | 使用此儀表板，可以監視 Web 伺服器的性能。它會以及時的方式顯示伺服器上的負載，以及其處理要求的能力。|使用「要求數」和`平均/最長要求時間`畫面可測量伺服器的總體繁忙程度。</br>尋找 URL 之間的相關性，以找到增強效能的機會。</br>使用「狀態碼」度量值可監視 4xx 和 5xx 錯誤碼。|  
| `HTTP 前幾名的要求`  |使用此儀表板，可以監視向 Web 伺服器要求次數最多的 URL。例如，要求總數、平均和最長要求處理時間，以及要求和回應中包含的資料流量。|   |
| `MongoDB`  | 使用此儀表板，可以監視 MongoDB 服務的忙碌程度、哪些集合處於最高需求狀態，以及哪些集合具有最慢效能。|尋找可受益於查詢和索引效能調整的集合。|
| `MySQL/PostgreSQL`  | 使用此儀表板，可以監視 SQL 資料庫交易的整體負載及效能狀態。找出要求數目，以及處理這些要求的速度。| 提升效能。例如，監視最慢查詢的回應時間，並識別您對查詢或表格上的索引所做的變更。|
| `MySQL/PostgreSQL 前幾名的查詢`  |使用此儀表板，可以監視要求次數最多的 SQL 查詢。檢視已接收的查詢數，以及針對查詢傳送及接收的資料流量。| 尋找最慢的查詢、執行最多時間的查詢，以及具有高資料流量的查詢。|
| `Nginx`  |使用此儀表板，可以監視主機資源、HTTP 連線數、最慢的 URL 以及主機回應碼。|透過監視下列度量值來識別負載平衡調整：寫入中、讀取中、等待連線、依機器的 CPU 負載、網路位元組活動、每秒要求數</br>查看「最慢 URL」度量值進行最佳化的頁面。</br>查看「回應碼」來尋找問題。|
{: caption="表 1. 應用程式及基礎架構元件的預先定義儀表板清單" caption-side="top"} 


## 主機與容器儀表板
{: #default_dashboards_host_container}

下表列出您可以用來在主機及容器中監視資源使用率及系統活動的預先定義儀表板。


| 儀表板名稱        | 說明                                                            | 用法             |
|-----------------------|------------------------------------------------------------------------|-------------------|
| `容器限制`    | 使用此儀表板，可以檢視與配置給每一個容器之主機資源相關的 CPU 和記憶體度量值。| |
| `檔案系統`         | 使用此儀表板，可以檢視裝載在實例上之檔案系統的目錄裝載點、檔案系統裝置，以及容量與使用情形資訊。</br>依預設，未列出遠端安裝的檔案系統。若要檢視這些檔案系統的資料，請將下列項目 **remotefs.enabled = true** 新增至每一個實例上的 */opt/draioss/bin/dragent.properties* 檔案。</br>選取群組時，度量值是類似文件系統裝載點的平均值。| 尋找哪些檔案系統即將填滿，以及哪些檔案系統未充分使用。依 *fs.bytes.used* 排序。|
| `依容器的概觀` | 使用此儀表板，可以檢視所選取群組或實例上執行之容器的資源使用統計資料，例如 CPU 百分比、記憶體用量百分比、網路位元組總數，以及檔案位元組總數。</br>在此視圖中使用*時光旅行* 特性，搭配 *Compare* 函數來查看歷程資訊。根據資料，驗證處理程序是否如預期執行、是否行為不當，或是否需要更多資源。| 找出哪些容器正在耗用大量資源。</br>請使用此資訊來判斷應用程式是否必須移至不同的主機。|
| `依容器映像檔的概觀` |使用此儀表板，可以取得容器映像檔中所定義服務的大小、效能和限制的概觀。| |
| `依主機的概觀` |使用此儀表板，可以檢視主機的一般資源用量統計資料，例如 CPU 百分比、記憶體用量百分比以及網路位元組總數。</br>您也可以使用此儀表板來監視實例的群組。</br>在此視圖中使用*時光旅行* 特性，搭配 *Compare* 函數來查看歷程資訊。根據資料，驗證處理程序是否如預期執行、是否行為不當，或是否需要更多資源。| 在具有類似工作功能的主機群組內，找出哪些主機未使用或過度使用。| 
| `依處理程序的概觀` |使用此儀表板，可以檢視主機上主要處理程序的一般資源用量統計資料，例如 CPU 百分比、記憶體用量百分比以及網路位元組總數。</br>您也可以使用此儀表板來監視實例的群組。</br>在此視圖中使用*時光旅行* 特性，搭配 *Compare* 函數來查看歷程資訊。根據資料，驗證處理程序是否如預期執行、是否行為不當，或是否需要更多資源。| 找出哪些處理程序正在耗用大量資源。</br>請使用此資訊來判斷應用程式是否必須移至不同的主機。|
| `Sysdig 代理程式摘要` | 使用此儀表板，可以檢視已在環境中部署的 Sysdig 代理程式數目及其版本。|  | 
| `前幾名的檔案` |使用此儀表板，可以檢視主要檔案的清單。您可以查看檔名、檔案位元組總數、檔案 I/O 中所花費的時間，以及檔案錯誤計數。| 依大小排序時，尋找前幾名的磁碟耗用者。</br>依 I/O 排序時，尋找最忙碌的檔案。</br>在*檔案錯誤計數* 直欄中識別監視資料時的潛在錯誤。|
| `前幾名的處理程序` |使用此儀表板，可以檢視在一個主機或一群主機執行之主要處理程序的清單。您可以查看處理程序名稱、其路徑及所有引數。</br>使用過濾特性可限制列出的處理程序。|在多次大量產生相同處理程序的環境中，尋找哪些處理程序是主要耗用者。|
| `前幾名的伺服器處理程序` |使用此儀表板，可以檢視識別為伺服器導向之處理程序（httpd、java、ntpd 等）的資源用量。| 尋找伺服器處理程序的資源使用。|
{: caption="表 2. 主機及容器預先定義儀表板的清單" caption-side="top"} 

## 網路
{: #default_dashboards_network}

下表列出您可以選取來監視網路連線及活動的預先定義儀表板：

| 儀表板名稱        | 說明                                                            | 用法             |
|-----------------------|------------------------------------------------------------------------|-------------------|
| `連線表`     | 使用此儀表板，可以檢視您的實例與遠端節點之間的連線。監視每一個連線的資料流量。| 尋找耗用網路上最多網路資料流量的主要發話者。|
| `概觀`              |  使用此儀表板，可以依方向、應用程式、處理程序及主機來檢視一段時間後的網路使用率總計。| 識別哪些資源最會影響回應效能。|
| `回應時間與資源用量` | 相較於處理網路要求中的主機回應時間，使用此儀表板可以檢視一段時間後的主機資源度量值。|  |
| `前幾名的埠`             |  使用此儀表板，可以檢視每個埠的送入位元組、送出位元組和位元組總數，以及顯示每個埠號的連線數目。|  |
{: caption="表 3. 網路預先定義儀表板的清單" caption-side="top"} 



## 服務
{: #default_dashboards_service}


下表列出您可以用來監視服務效能的預先定義儀表板，即使那些服務部署在編排的容器中也一樣：

| 儀表板名稱        | 說明                                                            | 
|-----------------------|------------------------------------------------------------------------|
| `依服務的概觀`   | 使用此儀表板，可以檢視在相同容器映像檔中執行之服務的大小、效能及限制。| 
| `服務概觀`      | 使用此儀表板，可以檢視單一服務的資源使用率及回應時間。| 
{: caption="表 4. 服務預先定義儀表板的清單" caption-side="top"} 



## 拓蹼
{: #default_dashboards_topology}

下表列出您可以用來監視應用程式層之邏輯相依關係的預先定義儀表板：

| 儀表板名稱        | 說明                                                            | 
|-----------------------|------------------------------------------------------------------------|
| `CPU 使用率`             | 使用此儀表板，可以監視主機與其餘基礎架構之間的互動。| 
|`網路資料流量`       | 使用此儀表板，可以監視本端處理程序與遠端節點上處理程序之間所選取實例的頻寬使用情形。|
| `回應時間`        | 使用此儀表板，可以監視所選取主機上所有處理程序之間的通訊及網路回應度量值。</br>回應計數和時間顯示為平均值，精確到秒。|
{: caption="表 5. 拓蹼預先定義儀表板的清單" caption-side="top"} 

**附註：**對於未安裝 Sysdig 代理程式但偵測到其通訊的實例，上述任何儀表板都會顯示虛線和灰色方框。
